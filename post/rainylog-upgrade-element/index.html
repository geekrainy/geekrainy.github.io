雨落

Posted on Sep 9, 2018

花了几天时间，将博客主题更新了下。一方面的原因是 NexT 主题已经烂大街了，作为 Hexo 最多 star 的主题，用的人多，且脱离 iissnan 的维护变为 theme-next 的组织维护后，变的越来越复杂。满足众多个性化需求的同时，又违背了简约的风格。原计划尝试下 hugo，了解了下其周边生态还是不如 Hexo 成熟。于是发现了 Element 主题，自己 fork 后稍微改了下，就用上了。有点骚又有点简单，挺喜欢的，目测又能用个一两年了。

此次改版也将博客名字由简单直白的 Rainy’s Blog 改为了雨落，用的是域名 rainylog 的字面音，也就意味着新的开始。

重新翻看了之前的文章，最早的一篇是 11 年写的。早期的文章来源于高中时代的 QQ 空间日志，原模原样搬运过来了。此后关闭 QQ 空间许久。随后进入了独立博客的时代，基本上流行的博客程序都被我折腾过一番，wordpress, emlog, z-blog, ghost, typecho 等等。也使用过点点网，Lofter 等第三方服务，奈何国内的服务逐渐都变了质，限制定制主题，限制绑定域名，慢慢也就放弃了。后来静态博客流行，就开始一直使用 Hexo 至今。如同高中折腾 Discuz 模板带我进入前端生涯一样，折腾 Hexo 也让我逐步开始参与开源项目，了解 Node，折腾 NexT 主题，从提一堆 issue 到主动参与维护，也让自己视野开阔了许多。接触了 github 的协作开发流程，受益匪浅。

学习技术的过程中，早期要有足够的广度。许多老师教育我们说，不要觉得当下学习的知识没用，说不定未来哪天就用到了，对我来说感触颇深。大学里自认没学到什么东西，倒是图书馆让我提升巨大。所有的计算机相关图书都在六楼，闲暇无事便去图书馆，随意翻看。最开始限于前端相关的书籍，随后又去探索没接触过的技术书。有编程语言，有软件工程，有服务器运维部署，还有操作系统和算法网络等等。带来的好处便是，对于后来工作需要去深入的知识，由于多多少少都接触过，学习起来也很快。有人会吐槽说这样每个方面都是浅尝辄止，没有深入。但如果目标定位于在技术领域走多年的路，现在为时尚早，不应将视野局限于某一处。令你在此处纠结的技术点也许在他处有了很成熟的解决方案，千万不能一叶障目。

转眼也已工作一年，很多同事都会很迷茫，面临抉择。对技术不自信，对未来不自信。要充分认识自己的不足，但也不能妄自菲薄，还是学到些东西的。自己感激工作上有一直持续学习的机会，工作上的进步不同于大学时期自学闭门造车，有外力推动，也有特定技术体系的标准化。这些是自学无法相比的。从毕业的大学同学里了解的情况是，基本步入工作的无一例外都对现状有或多或少的不满或不安。只想说，机会是自己创造的，很多人碌碌无为，也有很多人持续精进，愿大家都不忘初心。

计算机技术或者编程技术最初只是自己的折腾，作为一种爱好。然而，当下却也变为了自己的工作。性质上的转变或多或少都会丧失掉一定的兴趣驱动，而保持这种兴趣的方法便是始终保持有一个不熟悉的领域在折腾。之前搞数据库和服务器运维，那么就研究前端，如今搞前端，那么私下里就研究些如 Java 如算法等其他技能，这样始终都有一个兴趣，一种折腾的源泉伴随自己，总是会有所进步的。

当今知识付费总是在贩卖焦虑，信息的来源渠道也是各种各样，无法避免对技术新人的影响。作为技术学徒，很多人都会在网络营造的人才红海面前自卑，优秀的人很多，但也并非高不可攀。警惕技术上的见光死，许多大 V 的技术也许并不是那么好，很多名企的面试复杂然而进去也是螺丝钉。俗话说“面试造火箭，入职拧螺丝”，便是此意了。闲暇多去面试几家，看看不足，拥有反馈，调整自己目标计划，也是持续精进之道吧。PS: 自己也没有做到，接下来也去尝试 : )

唠叨完毕，继续 Coding。
